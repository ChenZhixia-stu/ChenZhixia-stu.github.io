7.Mysql数据库可以保存的数据类型

  • 整型：tinyint(-127~127)、int()

  • 浮点型：

    double(M,D) zerofill：

    ​ M：有效位数；

    ​ D：小数位数；

    double(4,2)：4位有效位数，2位小数；

    只会保留后两位，所以zerofill因版本原因，有时候会补零。

  • 日期型：

    Date：年月日

DateTime：年月日时分秒

TimeStamp：时间戳

Time：时分秒 hh:mm:ss

  • 字符型：

    char：如果有没有用完的长度，则用空格填满;检索速度快

varchar：可变长，只会占据实际的长度；可以占据更少的空间，检索(查询)速度稍慢

BLOB：把图片转换为16进制存储

CLOB：文本内容

  • 枚举型：

    ENUM:枚举类型,一次仅有一个值的字符串对象

  • 集合型：

    set

9.约束

约束是用来限制和保护表的数据符合我们定义的条件。

  1. not null 非空约束
  2. unique 唯一约束
  3. primary key 主键：非空唯一
  4. foreign key 外键

9.1建表的三大范式

  1. 确保每列的原子性。
  2. 确保表中的每列都要和主键列相关。
  3. 确保每个列都和组件有直接关系。

9.2表与表之间的关联关系

  1. 一对一 （1:1）：丈夫和妻子

外键随便放哪一方，但是只能放一方。

2.一对多（1：n）：飞机和乘客

外键列放在多的一方。

3.多对多（n:n）：老师和学生。使用桥表

外键放哪边都不合适，所以使用桥表；
