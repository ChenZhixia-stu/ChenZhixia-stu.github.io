Day4

1.软件安装完成。mysql,navicat
关系型数据库
mysql：数据库的一种，sql。以二维表的形式，持久化的保存数据。
操作：1.创建表 t_user
2.通过CRUD操作，来向表中添加数据
增：insert into t_user(id,name,age) values(10,”张三”,20);
删：delete t_user where id=10;
改：update t_user set name=”lisi” where id=10;
查：select 字段1,字段2,… from t_user;

注：1.每张表都有主键列，主键列必须是非空唯一的值。
   2.表和表之间的关联关系，是通过外键列来表示的。外键列的值可以为空，但是出现的话，必须是所依赖另一张表的主键列值所存在的值。


navicat：一种可以图形化展示数据库数据的一个软件。
         可以操作表，导出表结构以及表数据，生成E-R图等很多功能。

2.SpringBoot项目搭建成功！
基于SpringBoot项目，来学习Spring-data-jpa

什么是SpringBoot？
SpringBoot主要的功能是用来管理整个项目的，jar包的依赖通过maven仓库自动联网下载；框架常用的配置文件，SpringBoot大多数都提供了默认配置，我们可以简化配置文件的编写。
使用SpringBoot之后，我们可以把更多的精力放在编写代码上。

什么是Spring-data-jpa?
Spring提供的和数据库交互的一门技术，底层使用的是hibernate，特点：不需要我们去编写sql语句，只要按照框架规定的命名规则，框架会自动帮我们生成sql语句。
程序员需要做的就是创建 1.实体和映射关系；2.写生成sql语句的接口； 3.测试

SpringBoot的目录结构特点：
1.src/main/java
要将代码放到启动类的同包或者自包下面，这样SpringBoot才能统一管理
2.src/main/resources
存放的可以覆盖SpringBoot的默认提供的配置文件信息的一个文件，application.properties
3.src/test/kava
存放的是本地测试SpringBoot项目的测试类。

4.pom.xml
    存放的所有项目需要的依赖jar包以及插件。

Spring-data-jps操作步骤：

1.做映射（实体+关系）
java中的bean对象，通过注解，和数据库的表做映射。

2.编写dao层接口继承JpaRepository
2.1 自己的接口中，什么都不写，完全使用从父接口中继承的方法
2.2 编写自己的方法(方法名按照约定的规则)，jpa会根据约定自动生成对应的sql语句

​ 2.3 编写自己的方法（方法名不按照约定的规则），这时候需要我们提供语句（可以是sql语句，也可以是hql语句 ）

3.spring会自动提供dao层接口的实现对象，并且放入ioc容器中

4.哪些需要用到，就在哪里进行注入@Autowired

select id,name,age
from t_user
where id=6 and age = 20;

select id,name,age
from t_user
where id=6 or age = 20;
